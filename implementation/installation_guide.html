NeuraForge Installation Guide

Setup instructions for running the NeuraForge AI platform

Getting Started with NeuraForge

Welcome to NeuraForge! This guide will help you set up and run both the backend and frontend components of the NeuraForge platform on your local machine.

Python Backend FastAPI Next.js Frontend LLaMA 3.1

Prerequisites

Before you begin, make sure you have the following installed:

  • Python 3.9+ - Required for the backend server
  • Node.js 18+ - Required for the frontend application
  • npm - Node package manager (included with Node.js)
  • Ollama - For running LLM models locally
Note: This guide assumes you've already installed Ollama with the LLaMA 3.1:8b model. If you haven't, please visit Ollama's website for installation instructions.

Installation Steps

Install Node.js

If you don't have Node.js installed, download and install it from the official Node.js website.

To verify the installation, open a command prompt or terminal and run:

node --version
npm --version

You should see version numbers if the installation was successful.

Start the Backend Server

Navigate to the backend directory and run the setup script:

cd implementation\backend
setup_and_install.bat
run_server.bat

This will start the FastAPI server at http://localhost:8000.

Install Frontend Dependencies

Open a new command prompt or terminal window, navigate to the frontend directory:

cd implementation\frontend
npm install

This will install all the required dependencies for the Next.js frontend.

Start the Frontend Development Server

In the frontend directory, run:

npm run dev

This will start the development server at http://localhost:3000.

Access the Application

Open your browser and navigate to http://localhost:3000 to access NeuraForge.

Make sure both the backend (port 8000) and frontend (port 3000) servers are running simultaneously.

Troubleshooting

Backend Issues

  • Import errors: Make sure you've run the setup script which installs the local package
  • Ollama connection: Ensure Ollama is running with the LLaMA 3.1:8b model loaded
  • Port conflict: If port 8000 is already in use, you can modify the port in server.py

Frontend Issues

  • Node.js errors: Make sure you have a compatible Node.js version (v18 or higher)
  • API connection: Check that the backend server is running and accessible
  • Build errors: Try deleting the .next folder and node_modules, then reinstall with npm install

Next Steps

After successful installation, you can:

  • Experiment with different LLM models in Ollama
  • Modify the frontend UI in the src/components directory
  • Extend the backend API functionality in the neuraforge/api directory
